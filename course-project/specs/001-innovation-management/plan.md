# Implementation Plan: InnovatEPAM Portal вЂ” Phase 1 MVP

**Branch**: `001-innovation-management` | **Date**: 2026-05-15 | **Spec**: `specs/001-innovation-management/spec.md`

**Input**: Feature specification from `specs/001-innovation-management/spec.md`

## Summary

Build a local InnovatEPAM Portal MVP using Next.js 16 App Router, TypeScript strict mode, Tailwind CSS, shadcn/ui, Prisma ORM with SQLite, bcrypt for password hashing, and cookie/session authentication. The feature supports submitter registration, idea submission with one file attachment, submitter-specific idea views, admin review of all ideas, status lifecycle updates, and evaluation comments. No automated tests вЂ” manual testing only.

## Technical Context

**Language/Version**: TypeScript 5.x with Next.js 16 (App Router) and React 19

**Primary Dependencies**:
- `next`, `react`, `react-dom` вЂ” framework and rendering
- `tailwindcss`, `@tailwindcss/postcss` вЂ” utility-first styling
- `shadcn/ui` (components copied into `components/ui/`) вЂ” accessible UI components
- `prisma`, `@prisma/client` вЂ” ORM and SQLite schema management
- `bcrypt`, `@types/bcrypt` вЂ” password hashing (cost factor 12)
- `date-fns` вЂ” date formatting
- `uuid` вЂ” session token generation

**Storage**: SQLite database at `prisma/dev.db` via Prisma. File attachments stored in `public/uploads/` with UUID-prefixed filenames.

**Testing**: Manual testing only. No automated unit, integration, or E2E tests.

**Target Platform**: Local web browser (http://localhost:3000). No cloud deployment.

**Project Type**: Full-stack web application using Next.js App Router with API route handlers.

**Performance Goals**: Supports local development workflow. File uploads max 10MB. Page loads instant on localhost.

**Constraints**: Single-tenant local MVP. Admin account seeded, not self-registered. `public/uploads/` serves files as static assets. No protected download routing in Phase 1.

**Scale/Scope**: ~1 admin, up to 20 submitters for local demo purposes.
